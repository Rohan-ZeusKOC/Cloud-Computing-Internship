1. Azure App Service is a managed PaaS for hosting web apps, APIs, and mobile backends. Its main options are Web Apps for standard code, API Apps for RESTful services, and Web App for Containers for running Docker containers.  
2. Availability Sets group VMs within a single data center across separate fault and update domains to protect against local hardware failures. Availability Zones are physically separate data centers within a region that protect against entire data center outages.  
3. In IaaS (like Azure VMs), Microsoft manages hardware, while you manage the OS, runtimes, and apps. In PaaS (like Azure App Services), Microsoft manages the OS and runtime, leaving you to manage just the app and data. In FaaS (like Azure Functions), Microsoft handles everything including auto-scaling, and you only manage the code.  
4. Azure Virtual Network (VNet) isolates your cloud resources into Subnets. Azure Load Balancer handles Layer 4 traffic, while Azure Application Gateway provides Layer 7 routing. For connectivity, VPN Gateway offers encrypted internet tunnels, and ExpressRoute provides a private, dedicated connection.  
5. An Azure Storage Account is a secure cloud repository for data like blobs and files. You can access it through tools like the Azure Portal and Storage Explorer, via security methods like Access Keys and SAS tokens, or over the web using public or private endpoints.  
6. Scaling up means upgrading your App Service Plan to a higher tier to get more CPU and RAM for your existing instance. Scaling out means increasing the total number of VM instances running your app, which can be done manually or automatically via Autoscale rules to handle higher traffic.
